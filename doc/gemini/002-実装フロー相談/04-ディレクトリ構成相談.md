## ポケメモリア 最適ディレクトリ構成案（2025年版準拠）

```text
src/
├── app/                      # エントリーポイント・ルーティング
│   ├── actions/              # サーバーアクション（将来的なデータ保存用）
│   ├── api/                  # APIルート
│   ├── (auth)/               # 認証関連（Group化して管理）
│   ├── create/               # 履歴書作成ウィザード
│   │   ├── [step]/           # 動的ルーティングでステップを管理
│   │   ├── layout.tsx        # ウィザード共通デザイン（進捗バー等）
│   │   └── page.tsx          # /create の初期画面
│   ├── layout.tsx            # 全体共通レイアウト
│   └── page.tsx              # トップページ（LP）
├── components/               # 再利用可能な部品（Atomic Design）
│   ├── atoms/                # ボタン、入力欄、アイコン、ポケモンバッジ
│   ├── molecules/            # 入力フォームの1行、ポケモンカード（小）
│   ├── organisms/            # 作品選択グリッド、手持ち6匹リスト
│   └── templates/            # 履歴書の各スタイル（Paper/NeoGB/RetroRG）
├── hooks/                    # カスタムフック（useResume, useMobile 等）
├── lib/                      # ライブラリ設定（Gorm連携の定義や設定等）
├── utils/                    # ユーティリティ関数（日付整形、タグ生成 等）
├── styles/                   # テーマ設定、グローバルCSS
├── types/                    # TypeScript型定義（Resume, Pokemon, Title 等）
├── stores/                   # グローバルステート（Zustand等での状態保持）
├── constants/                # 固定値（ポケモン作品リスト、属性データ 等）
└── public/                   # 静的アセット（画像、フォント、ドット絵）

```

## 今回の見直しのポイント

### 1. Atomic Designによるコンポーネント管理

記事にある通り、`components/` 配下を `atoms/` から `templates/` までの4層に分割します。

* **メリット:** 「ボタンのデザインを直したい」と思ったら `atoms` を見ればいい、という直感的な作業が可能になります。

### 2. `stores/` の導入

ウィザード形式の「ページをまたぐデータ保持」には、記事でも触れられている **Zustand** 等のストア管理を `stores/` に配置するのが2025年的なベストプラクティスです。

* **理由:** `ResumeProvider` (Context) よりも記述がシンプルになり、1日1時間の開発スピードを落としません。

### 3. `app/create/[step]` による柔軟な遷移

各ステップを `/create/origin`, `/create/history` と個別に作るのではなく、`[step]` という動的セグメントで管理することで、ステップの追加や順序入れ替えが容易になります。

### 4. `templates/` に「3つのスタイル」を配置

仕様書にある **Authentic Paper** などのスタイルは、データの「入れ物」であるため、Atomic Designにおける `templates` として定義します。

## 1日1時間で進めるための「最初のコミット」対象

この構成を反映させるために、まずは以下のファイルを作成することから始めましょう。

1. **`constants/titles.ts`**: ポケモンの歴代作品データの定義（これがないと画面が作れません）。
2. **`types/resume.ts`**: 履歴書データの型定義（TypeScriptの恩恵を受けるため）。
3. **`stores/resumeStore.ts`**: ウィザードの入力を保持する器（Zustandなどのセットアップ）。
