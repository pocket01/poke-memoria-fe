# 質問
ありがとう。

> このユーザーフローに沿って、**「履歴書作成ウィザード（入力画面）」の具体的な項目とUI（入力しやすさ）**を深掘りしませんか？

こちらをお願いします。

# 回答
「履歴書作成ウィザード」の深掘りですね。
ユーザーにとって「入力」はもっともエネルギーを使う作業ですが、「ポケメモリア」においては**「自分の思い出を振り返る楽しい時間」**に変えることが理想です。

「人生の軌跡」を可視化するために、入力しやすさを追求したUI設計案を提示します。

---

## 1. 履歴書作成ウィザードの構成案

一気にすべてを入力させず、以下の4つのステップに分割します。

### ステップ1：あなたの原点（First Memory）

* **項目:** 初めてプレイした作品
* **UI:** 初代から最新作までを「パッケージ画像」または「カセットのアイコン」で横スクロール表示。
* **工夫:** 選択すると、その作品のイメージカラーに画面全体がふわっと色づく演出（没入感の向上）。

### ステップ2：歩んできた道（History）

* **項目:** これまで遊んだ作品（複数選択）
* **UI:** 作品年表（タイムライン）を表示し、プレイしたものをタップで「スタンプ」していく。
* **工夫:** 発売当時の思い出が蘇るよう、リリース年とハードウェア（GB, DS, Switch等）を併記。

### ステップ3：現在の相棒（Best Partners）

* **項目:** 好きなポケモン（最大6匹：1パーティ分）
* **UI:** 検索窓 + 予測補完。
* **工夫:** ポケモンの名前を入力すると、ドット絵や公式イラストが表示され、それを「モンスターボール」にドラッグ＆ドロップして登録する直感的なUI。

### ステップ4：あなたのスタイル（Trainer Identity）

* **項目:** 交流目的、プレイスタイル、フリーメッセージ
* **UI:** 「対戦」「収集」「色違い」「考察」などのタグをカード形式で選択。
* **工夫:** 最後は、自由記述の「意気込み」を記入。ここで、選んだビジュアルスタイル（紙ベースや赤・緑風）の「プレビュー」がリアルタイムで更新される様子を見せる。

---

## 2. 「入力しやすさ」を支えるUI/UXのポイント

| ポイント | 具体的な手法 |
| --- | --- |
| **オートフィル連携** | 最初に「初めて遊んだ作品」と「現在遊んでいる作品」を選ぶと、その間の世代を「プレイ済み候補」として優先表示する。 |
| **感情を揺さぶる演出** | 入力完了ごとに、ゲームのレベルアップ音や「回復の音」をSEとして鳴らす。 |
| **スキップの許容** | 「今は思い出せない」項目のために「あとで入力」ボタンを常設し、心理的ハードルを下げる。 |
| **ゲーミフィケーション** | 入力進捗率を「図鑑完成率」のように表示し、コンプリート欲を刺激する。 |

---

## 3. 具体的な画面イメージ（レイアウト）

選択した「紙ベース」や「赤・緑風」のスタイルに合わせた入力欄の工夫です。

* **「紙ベース」の場合:**
* キーボードで入力するのではなく、ペンで書き込むようなアニメーション。
* 選択項目は「チェックボックス」ではなく「〇で囲む」や「ハンコを押す」表現。


* **「赤・緑風」の場合:**
* コマンド選択画面（たたかう/にげる）のようなメニューUI。
* 文字入力もゲーム内のニックネーム登録画面をオマージュ。
