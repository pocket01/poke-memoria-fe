# 質問
> 1. 画面遷移図（ユーザーフロー）の作成: * アプリを起動してから、自分の履歴書を完成させ、他のトレーナーと繋がるまでの「操作の流れ」を整理します。
まずは1からお願いします。

# 回答
ビジュアルスタイルが決まったら、次はその「器」に魂を吹き込む**「ユーザーフロー（画面遷移）」**の設計です。

「ポケメモリア」の核である「軌跡の記録」と「トレーナー同士の繋がり」をスムーズに実現するための、標準的な画面遷移図案を作成しました。

---

## ポケメモリア ユーザーフロー（画面遷移図）

ユーザーがアプリをインストールしてから、目的を達成するまでの主な流れです。

```mermaid
graph TD
    A[起動・スプラッシュ] --> B{ログイン/登録}
    B --> C[プロフィール初期設定]
    C --> D[履歴書作成ウィザード]
    
    subgraph 履歴書作成（入力）
    D --> D1[ポケモン歴の選択]
    D1 --> D2[推し・相棒の設定]
    D2 --> D3[プレイスタイル設定]
    D3 --> D4[自由記述/軌跡の入力]
    end
    
    D4 --> E[マイ履歴書プレビュー]
    E --> F[ホーム画面]
    
    subgraph 繋がり・交流
    F --> G[おすすめトレーナー探索]
    F --> H[こだわり検索]
    G --> I[他ユーザーの履歴書表示]
    H --> I
    I --> J[お気に入り/繋がるリクエスト]
    end
    
    J --> K[マッチング/交流開始]

```

### 各画面の役割と設計のポイント

| 画面名 | 設計で意識すべきこと |
| --- | --- |
| **履歴書作成ウィザード** | 入力項目が多いと離脱するため、1画面1ステップで。紙ベースのデザインなら「書類に記入するワクワク感」を演出。 |
| **マイ履歴書プレビュー** | 選んだビジュアルスタイル（赤・緑風など）が適用された完成図を見せ、達成感を与える。SNSシェアボタンを配置。 |
| **こだわり検索** | 「特定の作品（例：クリスタル版）が好きな人」「ヒノアラシ推しの人」など、ニッチな条件で繋がれるように。 |
| **他ユーザーの履歴書** | 相手の「人生の軌跡」がドラマチックに見えるレイアウト。共通点（同じ作品を遊んでいる等）をハイライト表示。 |

---

## 精査すべき「最初の課題」

このフローを動かすために、デザイン着手時に決めておくべき重要なポイントが1つあります。

> **「履歴書の作成は、どのタイミングで完了させるべきか？」**

* **A. 最初にすべて入力:** 繋がりやすくなるが、登録のハードルが高い。
* **B. 最小限で開始し、後から肉付け:** 始めやすいが、最初はプロフィールがスカスカになりやすい。

「人生の軌跡」を重んじる「ポケメモリア」の場合、**最初は「初めてのポケモン」と「最新の推し」だけを聞き、残りはアプリを使いながら「思い出を綴る」ように少しずつ埋めていくフロー**が、ユーザーの負担を減らせるかもしれません。
