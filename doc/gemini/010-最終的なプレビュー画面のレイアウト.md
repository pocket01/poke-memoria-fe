# 質問
> ガイドの文言まで固まったことで、入力画面の設計がより具体的になりました。次は、**「入力した内容がどのように『履歴書』として1枚にまとまるか、最終的なプレビュー画面のレイアウト」**を検討しませんか？
お願いします。

# 回答
入力した「人生の軌跡」が、ついに一枚の「履歴書」として結実する**プレビュー画面（マイページ）のレイアウト**を検討しましょう。

この画面は、ユーザーが自分の歩みを誇らしく感じ、かつ他のトレーナーが「この人と繋がりたい！」と思うための最も重要な画面です。

---

## 1. プレビュー画面のレイアウト構成

スマホの縦長画面を活かし、スクロールすることで物語が展開するような構成にします。

| セクション | 内容と配置の意図 |
| --- | --- |
| **1. ヘッダー：アイデンティティ** | **トレーナー名とアイコン（相棒ポケモンとの2ショット）**を配置。ここで「誰の履歴書か」を即座に伝えます。 |
| **2. 軌跡のサマリー（勲章）** | **「プレイ開始年」「通算プレイ作品数」「メインスタイル」**をバッジのように並べ、一目で凄さや個性を伝えます。 |
| **3. メイン：ビジュアル履歴書** | 前段で選んだスタイル（紙ベース/赤緑風）で、**「ポケモン歴」が1枚の画像のように凝縮**されたエリア。SNSシェア用のメインビジュアルになります。 |
| **4. 好きの深掘り（推しポケ）** | **「好きなポケモン」を最大6匹**、こだわりのコメントや思い出（出会った場所など）と共にカード形式で表示。 |
| **5. フットプリント（自由記述）** | 履歴書の「志望動機」欄のように、**熱量の高いフリーテキスト**を配置。将棋などの他趣味もここに。 |

---

## 2. スタイル別・レイアウトのこだわり

### 【紙ベース】真面目な「公文書」風

* **質感:** スクロールするたびに紙が重なっているような重厚感。
* **レイアウト:** 罫線に基づいた整然とした配置。手書きの修正跡や、相棒ポケモンの写真がクリップで留められているような演出。
* **シェア:** 「PDF出力」や「証明書風画像」として書き出せるボタンを配置。

### 【赤・緑】レトロな「図鑑」風

* **質感:** 白黒ドットのコントラストを強調。
* **レイアウト:** ゲーム内の「つよさをみる」画面を拡張したような、ステータス画面風のUI。
* **シェア:** ゲームボーイの通信交換画面を彷彿とさせる「トレーナーカード」形式での書き出し。

---

## 3. 「繋がり」を生むためのUX機能

プレビュー画面には、自分が見るだけでなく「他人に見せる」ための以下の機能を盛り込みます。

* **「共通点」のハイライト:** 他人の履歴書を見た際、自分と同じ「初めての作品」や「好きなポケモン」が光ったり、強調されたりする機能。
* **ワンタップ・リアクション:** 「わかる！」「懐かしい！」「その相棒最高！」といった、ポケモン特有の感情を送れるスタンプボタン。
* **QRコード生成:** オフラインのイベント（対戦会や交換会）ですぐに自分の軌跡を提示できるよう、画面端に小さなQRコードを常駐。

## 最初のアクションとしての精査

プレビュー画面で最も「見栄え」を左右するのは、**「ポケモン歴（年表）をどう1画面に収めるか」**です。

* **案1（サマリー型）:** 全作品のロゴを並べるのではなく、「カントー済」「ジョウト済」のように地方ごとのスタンプにしてスッキリさせる。
* **案2（フルログ型）:** スクロール前提で、初代から最新作までを全て等間隔に並べ、密度の濃さをアピールする。

「人生の軌跡」という重みを出すなら、**案2のフルログ型をベースに、ユーザーが特に思い入れのある世代を大きく表示できる「ピックアップ機能」**を付けるのが良さそうですが、いかがでしょうか？
